01 · Problem & Background

Why this project exists

Childcare Culture & Co. (CCCC) has completed a brand repositioning under the banner "Culture Before Everything™." A six-page HTML design mockup was produced during the rebrand (Home, About, 90-Day Experience, Culture Club, Speaking, Contact) but the live website at childcarecultureco.com does not yet reflect the new brand.

The site needs to move onto the GoHighLevel (GHL) website builder inside the existing CCCC sub-account so that forms, booking, and payments run natively in the platform CCCC already operates on. This project is the build-out of that live site.

02 · Success Criteria

The project is done when all six are true

  1. All six pages are live on childcarecultureco.com, built in the GHL website builder, matching the approved brand system.
  2. Every form on the site submits into the CCCC GHL sub-account correctly (verified with test submissions).
  3. The booking widget connects to the 15-Min Strategy Call calendar and test bookings land correctly.
  4. Both membership products have their own separate, working payment checkout, verified with test transactions: The Culture Club — $99/mo and The 90-Day Culture Transformation Experience™ — $297/mo.
  5. The site is fully responsive (mobile, tablet, desktop) with no broken layouts.
  6. Anj has signed off on each phase (see Section 07) and on final launch. Nothing is published until sign-off.
03 · Scope

What's in, what's untouchable

IN SCOPE
  • Six pages in the GHL website builder: Home · About · The 90-Day Culture Transformation Experience™ · The Culture Club · Speaking · Contact
  • Design direction: the six-page HTML mockup is the reference, not a pixel-perfect spec. Mark may refine layout and design details, but the brand system is locked (Section 05)
  • Forms wired into GHL: general contact form (Contact page), speaker inquiry form (Speaking page — separate from the contact form, per mockup), and any opt-in/lead-capture forms present in the mockup
  • Booking widget connected to the existing 15-Min Strategy Call GHL calendar — used for all "Schedule Your Culture Strategy Call" CTAs
  • Two separate payment products/checkouts (Stripe already connected): The Culture Club ($99/mo recurring) and The 90-Day Culture Transformation Experience™ ($297/mo recurring)
  • Redirect handling: a live "Culture Collective" campaign points a bit.ly link at a page on the current site — that destination must redirect to the 90-Day Experience page at launch so the campaign link doesn't 404
  • Basic on-page QA: responsive testing, link checking, form/booking/payment test runs
DO NOT TOUCH YET
  • GHL workflows, automations, pipelines (including the CCCC sponsorship pipeline), tags, contacts, or email campaigns. The build must not create, edit, or delete any of these except the specific forms/calendar/payment connections listed in scope
  • Domain/DNS — childcarecultureco.com is on GoDaddy and already connected to GHL. No DNS changes
  • The Stripe integration itself — already connected and working. Mark creates products/checkouts on top of it but does not alter the integration
  • Summit 2027 marketing assets, registration pages, or funnels — the Summit nav item links out to the existing page at childcarecultureco.com/bustamove
  • Blog, SEO campaign work, or analytics setup — can be scoped as a later phase if wanted
  • Copywriting from scratch — mockup copy is the source of truth; copy changes go through Anj

06 · Technical Constraints

How it must be built

  • Platform: GHL website builder only. No external hosting, no custom-coded standalone site.
  • Custom code blocks (HTML/CSS embeds) inside the GHL builder are allowed where the native builder can't achieve the design, but should be the exception, not the default — they're harder to maintain after handoff.
  • Image assets are already uploaded to GHL Media Storage. Mark works from those and flags any missing assets to Anj rather than sourcing his own.
  • Payments: two separate products built on the existing Stripe connection.
Hard rule

Nothing goes live without sign-off. All pages are built unpublished/staged. The current live site stays untouched until Anj approves launch, at which point the new pages replace it on the same domain.
